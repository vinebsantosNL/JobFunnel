KYC — Onboarding redesign

Tags Project Delivery Prioritization Stakeholder Mgmt ~320 words · Created 17 Mar 2026
S Situation

At my previous fintech company, we had a 68% drop-off rate during the KYC (Know Your Customer) onboarding flow. Users were abandoning the identity verification step because it required too many document uploads and had confusing error states. This was directly impacting our activation rate and costing us approximately €40k/month in lost conversions.

T Task

As the PM owning onboarding, I was tasked with redesigning the KYC flow to reduce drop-off by at least 20% within one quarter, without compromising compliance requirements or adding engineering scope beyond 2 sprints.

A Action

I ran 8 moderated user sessions to map where confusion occurred, then partnered with our compliance team to understand the minimum viable document requirements. I redesigned the flow in Figma with three key changes: progressive disclosure (step-by-step instead of all-at-once), clearer error recovery states, and a "save and continue later" option. I also wrote the PRD, led the 2-sprint build, and tracked daily drop-off metrics via Amplitude to catch regressions early.

R Result

KYC drop-off fell from 68% to 34% within 6 weeks of launch — a 50% improvement, double our target. Monthly revenue recovered approximately €22k of the previously lost conversions. The "save and continue" feature alone accounted for 18% of completions. The redesign became the template for our subsequent market expansion flows in Germany and the Netherlands.

Story Library — Product Analysis

Market analysis · Problem framing · UI prototype · PRD

1

Market Analysis — How Outstanding SaaS Manage Content Libraries

The core problem of Story Library is a gallery management + coverage tracking challenge. The closest analogues in the SaaS world are knowledge base tools, content libraries, and portfolio managers. Here's how the best solve it:

N
Notion — Database Views
Most relevant

Notion solves gallery management by offering multiple views of the same dataset: List, Table, Board (Kanban), Gallery, Calendar, and Timeline. Users choose the mental model that fits their use case. The Gallery view shows rich card previews with cover images and key properties visible. The Filter + Group-By system lets users slice any way they need.

Multi-view (list/grid/board) Property filters Group-by any field Card previews

→ JobFunnel steal: Multi-view toggle (List / Card / Coverage Matrix)

L
Linear — Issue Library
UI patterns

Linear's issue list excels at richness without noise: each list item shows status, priority, assignee, labels, and a one-line description — all scannable in a single row. Keyboard-first navigation (J/K to move, N to create), instant search, and label-based grouping make it feel fast even with hundreds of items. The completeness of a ticket (has description? has estimate?) is visible via subtle icons.

Rich list items Keyboard-first Completeness signals Instant filter

→ JobFunnel steal: STAR completeness badge (4/4), incomplete warning inline

A
Airtable — Gallery Base
Layout

Airtable's Gallery view renders records as visual cards with an image or color header, making the content scannable at a glance. The grouping feature (Group by field) is the key insight: users can view all records grouped by category, seeing the distribution. An empty group explicitly shows "0 records" — which is a gap signal.

Color-coded headers Group-by category Empty group = gap signal

→ JobFunnel steal: Coverage Matrix view with explicit 0-story gap prompts

P
Pitch — Slide Library
Gallery UX

Pitch manages a gallery of reusable slide templates and decks. Key patterns: visual thumbnail preview (user understands content without clicking), tag-based organization, and usage count on each template ("used 12 times"). This usage signal tells users what's proven vs. untested.

Usage count Rich preview Tag organization

→ JobFunnel steal (Phase 2): "Used in 3 interviews" badge when linked to applications

I
Interviewing.io / Exponent — Story Banks
Direct competitor

Interview coaching tools like Exponent have "story banks" where users save STAR answers by question type. Their approach: question-first (e.g., "Tell me about a conflict you resolved") → link to saved stories. The gap here: they don't tell users which competency areas are uncovered, they just list stories linearly. No coverage analytics.

Question-linked stories No coverage analytics ✗ No gap detection ✗

→ JobFunnel advantage: Coverage matrix + gap detection is a genuine differentiator

Key insight from market analysis:

No tool at the intersection of interview prep + content management currently offers competency coverage analytics. Notion-style multi-view + Linear-style completeness signals + Airtable-style gap visibility = a story library that actively coaches the user toward interview readiness, not just stores content.

2

How This Problem Can Be Solved — 6 Design Bets

1

Multi-view: List / Card / Coverage Matrix

Different users need different mental models. List = scan quickly. Card = rich preview for low volume. Coverage Matrix = the strategic view that tells you "am I interview-ready?" — something completely absent today. The view should be persistable per user.

Effort: M · Impact: H

2

STAR Completeness Indicators

Every story should surface how complete it is (0/4, 1/4, 2/4, 3/4, 4/4 STAR sections filled). Incomplete stories are interview liabilities. The indicator should appear in every view, and incomplete stories should have a "Complete this story →" CTA in the detail panel. This is a Linear-inspired UX pattern applied to interview prep.

Effort: S · Impact: H

3

Inline Gap Prompts — Coaching in Context

When a user has 0 stories in a competency, surface a contextual prompt inside the list ("No Growth stories yet — senior roles often ask about learning agility") with a direct "+ Add story" CTA. This turns the library from passive storage to an active interview prep coach. The prompt should be dismissable but reappear after 7 days.

Effort: S · Impact: H (retention + engagement)

4

Richer Story List Items (Information Density Upgrade)

Current list items show: title, 1 tag, date. This is too sparse for a library. Add: category (colored dot + label), excerpt (2 lines), STAR completeness badge, word count (proxy for story depth), favorite. This gives the user enough signal to pick the right story without opening it.

Effort: S · Impact: M

5

New Story Form: Guided Prompt + Gap Awareness

When creating a new story, show a banner if the user has uncovered gaps: "You have no Collaboration stories — consider adding one." The competency picker should visually highlight uncovered categories. Field labels should include contextual hints ("Situation: Set the scene. What was happening?") to scaffold good writing.

Effort: S · Impact: M

6

(Phase 2) Usage Tracking — "Used in X interviews"

Once stories can be linked to job applications, surface a "Used in 3 interviews" badge on each story. This tells the user which stories are battle-tested vs. untested. Proven stories get a subtle green highlight. Stories never used get a "Never used — try it in your next interview" nudge.

Effort: M · Impact: H · Phase 2

3

PRD — Story Library v2

Problem Statement

The Story Library currently functions as passive storage — users can add and retrieve stories, but the product does not help them understand whether they are interview-ready. There is no signal for story completeness (partial STAR vs. full STAR), no coverage visibility (which competency areas are missing), and no coaching prompts. The result: users store 1–3 stories and stop adding more because the product does not show them the value of a comprehensive library.

Goals

  • Increase average stories per user from ~2 to 5+ within first 30 days
  • Increase story completeness rate (4/4 STAR) from unknown to measurable baseline >60%
  • Create a coverage matrix that becomes a Pro feature differentiator
  • Reduce time to find the right story before an interview (<10 seconds)

Features — Prioritized

Feature Priority Tier Effort
STAR completeness badge on list items P0 Free + Pro XS (1d)
Richer list item (category dot, excerpt, word count) P0 Free + Pro S (2d)
Inline gap prompts in list (per missing competency) P0 Free + Pro S (2d)
Card / Gallery view mode P1 Free + Pro M (3d)
Coverage Matrix view (bar chart per category) P1 Pro only M (3d)
New story form: gap awareness banner + field hints P1 Free + Pro S (1d)
View preference persistence (Zustand + localStorage) P2 Free + Pro XS (0.5d)
Story interview usage tracking (Phase 2) P3 Pro only L (requires schema)

Schema Changes Required

-- No schema change for P0 features

-- Completeness is computed from existing columns (situation, task, action, result)

-- Gap detection is derived from competencies[] array and user's full story set

-- Phase 2: story_usages table (links story_id → job_application_id)

CREATE TABLE story_usages (

id UUID PRIMARY KEY,

story_id UUID REFERENCES interview_stories(id) ON DELETE CASCADE,

job_application_id UUID REFERENCES job_applications(id) ON DELETE CASCADE,

user_id UUID REFERENCES profiles(id),

used_at TIMESTAMPTZ DEFAULT now()

);

Acceptance Criteria (P0 + P1)

  • ✓ Each story item in the list displays: category dot + label, excerpt (2 lines), STAR completeness badge (n/4), word count, date, favorite toggle
  • ✓ Completeness badge: green "4/4 ✓" for complete, amber "n/4" for partial, red "0/4" for empty (freeform stories show "Full" badge)
  • ✓ At most 1 gap prompt visible in list at a time (for the highest-priority missing competency). Dismissable, reappears after 7 days.
  • ✓ View toggle (List / Card / Matrix) persists to localStorage via Zustand middleware
  • ✓ Card view: renders color-coded top border (category color), title, excerpt, completeness, word count, date. Click opens detail panel.
  • ✓ Coverage Matrix: 1 bar per competency, filled proportionally, shows covered sub-tags (✓) and uncovered sub-tags (gray). Overall completeness score shown at bottom.
  • ✓ Coverage Matrix is behind ProGate for free users — shows blurred preview + upgrade CTA
  • ✓ New story form shows blue banner with uncovered gaps when user has ≥1 gap
  • ✓ All views are mobile-responsive: Card view wraps to 1 column at <640px, Coverage Matrix scrolls horizontally never — stacks vertically
