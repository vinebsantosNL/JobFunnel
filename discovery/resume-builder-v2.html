📊 Screening Rate Badge
Calculated as Screenings ÷ Applications for jobs tagged with this CV version in the pipeline.
  • Green — rate ≥ 18% (at or above EU tech average)
  • Amber — rate 8–17% (below average, coach user)
  • Gray — fewer than 10 applications (low confidence)
Clicking the badge navigates to Performance tab filtered to this version.
🛡️ ATS Safety Badge
Computed from a rule-based check on the CV's structured data:
  • Single-column layout → pass
  • Standard section headers → pass
  • No images or skill bars → pass
  • Contact not in page header/footer → pass
ATS Safe = all 4 pass. Review needed = 1+ fail.

Clicking navigates to ATS Score tab for detail.
⭐ Default CV Version
The default version is pre-selected when adding a new application in the Pipeline. Users can override per application in the job detail sheet.

Only one version can be default at a time. Setting a new default unsets the current one. Default status is shown with a blue border on the card.
🔒 CV Version Lock Rule
A CV version is editable only while applications are in Saved or Applied stages. Once any linked application reaches Screening, the version is locked for editing to preserve the attribution integrity for analytics.

The API returns 422 if a locked version is edited. The UI shows a lock icon and tooltip explaining why.
🔍 Pipeline Keywords
Keywords are extracted from job descriptions of applications in your pipeline tagged "Saved" or "Applied". They are ranked by frequency across those JDs.

Missing keywords = high-frequency terms not found in this CV version's structured data fields. Clicking a keyword shows which pipeline jobs mention it and suggests which CV section to add it to.
📄 PDF vs DOCX Export
The system detects which ATS platforms are used by companies in the user's active pipeline and surfaces a contextual recommendation:
  • Greenhouse / Lever → PDF is safe and preferred
  • Workday → DOCX is safer (PDF has partial support)
  • SAP SuccessFactors / Taleo → DOCX only
ATS detection is based on a curated database of company ↔ ATS mappings, not user input.
